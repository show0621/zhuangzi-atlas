# 莊子全解（Zhuangzi Atlas）— Agent 工作守則

你正在協助撰寫與維護出版級專案《莊子全解》。目標是：**一本書 × 一個網站 × 一套 AI 知識庫**，品質介於王邦雄《莊子內七篇‧外秋水‧雜天下的現代解讀》、陳鼓應《莊子今註今譯》、傅佩榮《傅佩榮解讀莊子》之間，並加入完整知識架構與現代應用。

## 專案身分

- 中文名：莊子全解
- 英文名：Zhuangzi Atlas
- 副標題：原典・白話・哲學・人生智慧
- 當前版本：V1.0（審定／出版稿）

## 版本路線（不可跳步吹噓完成）

| 版本 | 範圍 |
|------|------|
| V0.1 | 專案骨架、網站、電子書流程、33 篇範本 |
| V0.2 | 導論 + 內篇 7 篇實質內容 |
| V0.3 | 外篇 15 篇 |
| V0.4 | 雜篇 11 篇 |
| V1.0 | 出版版：索引、百科、思想地圖、交叉引用齊備 |

## 內容分層（強制）

撰寫任何解析時，必須清楚區分：

1. **原典**：引用《莊子》原文（標篇名與版本依據）
2. **歷代注解**：郭象、成玄英、林希逸等（標注家名）
3. **本書現代詮釋**：哲學分析與人生應用（明標為詮釋，不可偽托原文）

禁止把注家說法或個人心得寫成「莊子說」。

## 篇章固定結構（33 篇統一）

每一篇 Markdown 必須包含以下標題（順序固定）：

1. 篇名與背景
2. 成書背景
3. 結構分析（含結構圖）
4. 原典（必要且合理引用；含「原典位置」）
5. 白話翻譯
6. 字詞註解
7. 段落解析（說明為何這段在這裡、與上下文關係）
8. 歷代注家怎麼看
9. 哲學分析
10. 與老子比較
11. 與儒家比較
12. 與佛學比較（交代後世並讀與不宜混同；禁止硬比附、禁止說成佛學、禁止「暫略」了事）
13. 現代人生應用
14. 常見誤解
15. 本篇總結
16. 心智圖
17. 延伸閱讀

詳細提示詞見 `prompts/chapter-template.md`。

## 寫作風格

- 貼近**原文脈絡與順序**，不以主題拼貼取代篇章結構。
- 嚴謹、可出版；避免空泛心靈雞湯。
- 現代應用必須**回扣原文概念**（如無待、心齋、無用之用）。
- 初學者可讀，進階者有注腳／延伸閱讀可深入。
- 使用繁體中文（台灣用字習慣為主，專名依通行學術用法）。

## 檔案約定

- 篇章：`content/chapters/{00-導論|01-內篇|02-外篇|03-雜篇}/`
- 人物：`content/figures/`
- 名詞：`content/terms/`
- 主題：`content/themes/`
- 思想地圖：`content/maps/`
- 書目：`content/references/`
- 目錄真相來源：`src/lib/catalog.ts`

更新篇章狀態：`skeleton` → `draft` → `review` → `published`（frontmatter 與 catalog 盡量同步）。

## 工程約束

- 不要擅自改 `output: "export"` 除非要做伺服器端「莊子 AI」。
- 不要提交 secrets、`.env`、個資。
- 不要大規模重寫未完成章節的骨架標記，除非使用者要求。
- 新增篇章前先改 `src/lib/catalog.ts`，再跑 scaffold／手動建檔。
- 完成實質內容後執行：`npm run content:validate`、`npm run indexes:build`。

## 每次寫作任務的預設流程

1. 確認目標篇章與目前 `status`
2. 讀取該篇 Markdown 與 `prompts/chapter-template.md`
3. 依原典順序撰寫／修訂，區分三層聲音
4. 補交叉引用（人物／名詞／主題／他篇）
5. 更新 `status` 與 `updated`
6. 跑 validate（若環境可用）

## 莊子 AI（未來）
